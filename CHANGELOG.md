# Changelog

## Unreleased

## v1.2.31

- 优化播放常驻定时器：移除永久运行的 `280ms setInterval`，改为仅在真实播放状态下自调度单个 `setTimeout`。
- 暂停、播放结束、空队列和普通隐藏状态会立即清除播放 tick，不再每秒多次唤醒主线程做空判断。
- 播放开始或页面恢复可见时立即重启 tick；“直播后台保持”开启后隐藏播放仍维持原 `280ms` 听歌统计和进度刷新语义。
- 连续 `play` / `playing` 事件共享同一计时任务，不会叠加多个进度循环。

## v1.2.30

- 修复长本地音频时长误判：本地 `song.duration` 明确按秒处理，`durationMs` / `dt` 明确按毫秒转换，20 分钟以上歌曲、长录音和有声内容不再被错误缩短。
- 修复非有限时长回退：`duration=Infinity` 或无效值时继续使用可用的毫秒字段，避免详情时长、码率估算和进度上限归零。
- 优化进度文本刷新：播放循环只在整数秒或总时长变化时重新格式化时间文本，同一秒内不再重复创建相同字符串。
- 优化进度节点恢复与 Media Session：DOM 节点重建后强制补写进度，系统媒体位置在节流命中时提前返回，减少无效时长/位置读取。

## v1.2.29

- 优化歌词字距测量与绘制：按 Unicode code point 游标扫描文本，不再为每次歌词贴图绘制创建 `Array.from` 字符数组，Emoji 和扩展汉字语义保持不变。
- 优化进度条拖动热路径：拖动开始时只读取一次轨道布局，后续 `pointermove` 复用矩形，减少连续拖动时的强制布局读取。
- 修复多指针和非主键误操作：拖动只接受发起指针，忽略右键及其他指针，零宽轨道和切歌竞态不再留下异常拖动状态。
- 优化拖动粒子清理：每轮三个粒子通过 `DocumentFragment` 单次插入，并合并为一个清理任务，视觉数量和动画保持不变。

## v1.2.28

- 优化列表副标题与本地音质缓存键：队列签名、搜索结果、左侧曲库和 3D 歌单架热路径改为直接字符串拼接，减少临时数组分配。
- 优化 3D 歌单架卡片绘制签名与队列采样签名：rebuild/redraw 判重路径不再为每张卡或每个采样点创建 join 数组。
- 优化队列渲染指纹、本地曲库面板签名和本地快照/索引签名生成：大列表刷新时改为单次字符串累加。
- 优化播放进度时间格式化：`timeupdate` 热路径去掉 `padStart` 临时字符串包装。
- 优化引导尾迹裁剪：长度和过期点用批量 `splice` 清理，避免循环 `shift` 搬移数组。

## v1.2.27

- 新增 `268 × 58` 极简迷你播放器：不显示或加载歌曲封面，保留歌曲名、歌手、上一首、播放/暂停、下一首和返回主界面。
- 原有 `360 × 84` 标准迷你播放器页面与视觉保持不变；设置面板和托盘菜单可在“标准（带封面）/ 极简（无封面）”之间切换。
- 两种迷你播放器分别保存拖动位置，重启和显示器工作区变化后按对应尺寸恢复；切换样式时立即重建当前迷你窗口。
- 极简模式的主进程到迷你窗口状态补丁不携带封面字段，避免跨进程复制无用的大型封面 data URL。

## v1.2.26

- 修复迷你播放器渲染进程连续崩溃时无限 `reload()`：单个窗口只尝试一次重载，未成功前再次崩溃会销毁旧窗口并按现有流程重建；成功加载后重置次数。
- 优化迷你播放器锁屏后台占用：Windows 锁屏或系统休眠时停止 5 秒恢复定时器，解锁或唤醒后沿用现有恢复流程重新校正可见性。
- 修复迷你播放器 IPC 状态误确认：主进程返回 `ok:false` 或旧完整请求晚于新状态请求失败时，会按补丁字段失效本地同步缓存，下一次事件重新发送缺失信息。
- 优化迷你播放器播放状态同步：连续 `play` / `playing` / `pause` 事件只检查播放状态，不再重复解析当前歌曲和封面签名；首次同步、切歌及元数据变化仍自动发送完整信息。
- 完善迷你播放器位置恢复：拖动结束后保存坐标，重启继续使用；系统 `setBounds()` 校正不再误判为用户拖动，显示器变化后的工作区坐标会同步更新。
- 优化迷你播放器拖动持久化：改用 Windows `will-move` / `moved` 事件区分手动与程序移动，同一坐标不重复写入设置文件。
- 优化迷你播放器恢复循环：窗口已正常显示且仍为置顶时，只刷新 Z 序，不再每 5 秒重复调用显示、写入置顶状态或构建状态同步补丁。
- 修复迷你播放器空队列仍保留旧封面数据、重复解析当前元数据的问题；封面加载失败后会释放失效地址，后续状态同步可重新尝试加载。
- 优化无歌词占位检测：LRC/YRC/自定义歌词过滤时改用单次字符扫描压缩文本，减少正则 replace 中间字符串。
- 修复迷你播放器运行一段时间后仍存在但被其他窗口覆盖、再次关闭主界面也无法重新显示的问题；显示时会重新确认原生置顶状态并恢复可见层级。
- 增加迷你窗口隐藏、最小化、显示器变化、系统解锁/唤醒和渲染进程异常后的自动恢复；旧窗口回调不再误清理新窗口实例，主界面和迷你模式可反复切换。

## v1.2.25

- 新增可开关的紧凑迷你播放器：主窗口最小化或关闭后显示 `360 × 84` 小窗，提供封面、歌曲名、歌手、上一首、播放/暂停、下一首和返回主界面。
- 迷你播放器支持拖动位置、置顶显示和多显示器工作区校正；恢复主界面时自动隐藏，不改变原有主播放界面布局与视觉系统。
- “桌面 / 壁纸”和托盘菜单新增同步开关并由主进程持久化；关闭开关后不再创建迷你窗口，开启时关闭按钮直接进入迷你模式。
- 播放命令复用现有播放器状态机，歌曲、封面和播放状态采用签名判重与增量 IPC 同步，不使用常驻轮询，暂停/继续时不重复传输整张封面。

## v1.2.24

- 优化本地歌词/文本解码：复用已有 TextDecoder 缓存，并用单次循环统计 U+FFFD，避免乱码判断时创建正则 match 数组。
- 优化 YRC 逐字歌词解析细节：前导空白长度改为轻量计数，减少逐字歌词行处理中的临时数组。
- 优化音量拖动热路径：音量与界面仍即时响应，偏好写入改为停手后合并持久化；百分比、静音状态和音量 SVG 只在结果变化时更新，减少连续拖动时的同步存储和 DOM 重建。
- 优化播放会话保存：常规进度保存移到浏览器空闲时段，退出时强制落盘，清空队列时取消待写任务，保持恢复曲目和进度语义不变。
- 优化播放状态同步：播放图标、控制栏节点、歌曲标题/副标题和 Windows Media Session 元数据增加缓存与判重，连续 `play/playing` 等事件不再重复写相同状态。
- 优化桌面歌词与封面后台路径：复用当前歌曲元数据和桌面歌词签名缓冲区，直接写入节奏图载荷并减少封面签名临时数组，降低桌面歌词高帧率同步及队列刷新时的短命分配。
- 本轮只调整内部歌词/文本解析、持久化、状态同步、缓存和签名路径，不改变 UI、布局、文案、视觉质感、歌词效果、播放入口或 3D 歌单架交互。

## v1.0.0

- 调整播放器控制台显隐规则：播放、切歌、天气电台开播不再主动弹出底部控制台；只有底部手柄/热区唤出时显示，鼠标移出热区后快速收起。

- 修复网易云 / QQ 扫码成功但登录态不落地的问题：网易云不再因资料接口短暂无 profile 立刻清空 cookie，QQ 登录支持新式 `psrf_*` / `wx*` token。
- 修复播放/暂停按钮手动恢复播放失效：手动点击路径现在会在用户点击同步栈内先调用 `audio.play()`，再恢复音频分析器，避免错过浏览器用户激活窗口。
- 修复播放准备阶段的栈溢出会拖死控制台的问题：歌词预置、封面加载、喜欢状态、Home 状态、listen session 和 beatmap / DJ 视觉预热均改为隔离执行，任何非音频链路异常不再阻断真实切歌。
- `playQueueAt` 增加播放阶段日志；若后续仍出现异常，控制台会标明 `source-url`、`visual-prep`、`audio-start` 等阶段，界面不再直接弹出 `Maximum call stack size exceeded`。
- 单曲循环的 `ended` 回调改为异步再入，避免浏览器在音频结束事件中同步重进 `playQueueAt` 引发递归风险。
- 修复天气电台 / 歌单播放链路的控制台递归风险：不再运行时重写播放、上一首、下一首按钮的 `onclick`，避免 `Maximum call stack size exceeded`。
- 切歌开始时立即刷新队列高亮和迷你队列计数；即使浏览器拦截自动播放，标题、时长、队列当前项也保持同步。
- Home 天气电台主按钮和天气卡片统一到同一启动入口，并增加启动 busy 锁，避免重复点击并发开播。
- 修复右键唤起 3D 歌单架时误唤出播放器控制台：歌单架打开期间会硬隐藏底部控制台，并阻止热区/播放态 reveal。

- 隔离 3D 歌单架刷新/关闭异常：歌单 API 成功后不再因为 shelf 重建或 3D 详情列表绘制失败而误报“歌单加载失败”。
